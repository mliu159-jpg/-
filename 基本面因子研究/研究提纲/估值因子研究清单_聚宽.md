# 估值因子研究清单（聚宽平台）

> 五大类基本面因子之一。估值因子衡量"价格相对基本面是否便宜"，是"深度价值"策略最核心的一类。本清单覆盖估值因子的完整族谱、聚宽数据来源、计算方式、因子方向与回测注意事项。

---

## 一、因子总览

| 因子族 | 核心因子 | 聚宽来源 | 预期方向 | 说明 |
|---|---|---|---|---|
| 市盈率族 | PE / EP | `valuation.pe_ratio` | EP 越大越好 | TTM 口径，反映盈利定价 |
| 市净率族 | PB / BP | `valuation.pb_ratio` | BP 越大越好 | 经典价值因子（Fama-French HML） |
| 市销率族 | PS / SP | `valuation.ps_ratio` | SP 越大越好 | 适合亏损/周期股 |
| 市现率族 | PCF / CFP | `valuation.pcf_ratio` | CFP 越大越好 | 经营现金流口径，抗盈余操纵 |
| 企业价值族 | EV/EBITDA、EV/Sales、EV/EBIT | 需自算（balance + income + cash_flow） | 倒数越大越好 | 含负债，跨资本结构可比 |
| 股息族 | 股息率 | `finance.STK_XR_XD` 计算 | 越大越好 | 深度价值与红利策略的交集 |
| 复合 / 相对估值 | PEG、估值历史分位、估值同比变化 | 自算 | 见各项说明 | 把"成长"和"时间维度"引入估值 |

「预期方向」指在 A 股价值效应下、长期看大概率为正的方向，实际仍需用 IC 逐期验证。

## 二、聚宽数据获取基础

### 估值表 `valuation` 字段（已验证）

| 字段 | 含义 | 口径 |
|---|---|---|
| `valuation.code` | 股票代码 | — |
| `valuation.pe_ratio` | 市盈率 PE | TTM（动态） |
| `valuation.pb_ratio` | 市净率 PB | 最新 |
| `valuation.ps_ratio` | 市销率 PS | TTM |
| `valuation.pcf_ratio` | 市现率 PCF | TTM，经营现金流 |
| `valuation.market_cap` | 总市值 | 单位：**亿元** |
| `valuation.circulating_market_cap` | 流通市值 | 单位：**亿元** |
| `valuation.turnover_ratio` | 换手率 | 当日 |

`valuation` 表是按交易日组织的，比率分子用当日收盘价、分母用当时已公告的财务数据，所以直接取这几个字段相对不容易踩未来函数。另有静态口径的 `pe_ratio_lyr`（最近年报 PE），研究上一般用 TTM 更及时。

### `get_fundamentals` 基本用法

```python
q = query(
    valuation.code,
    valuation.pe_ratio,
    valuation.pb_ratio,
).filter(
    valuation.code.in_(stock_list)   # 注意：用 in_()，不能用 in
)
df = get_fundamentals(q, date='2024-12-31')
```

三个要记住的点：筛选用 `in_()` / `or_()` 而非 Python 原生 `in`；财务三大表（income / balance / cash_flow）在聚宽是**单季度值，不是累计值**，算 TTM 要自己滚动求和；`valuation` 表的比率字段平台已算成 TTM，可直接用。

### 另一条路：聚宽因子库

`from jqfactor import get_factor_values, get_all_factors` 提供数百个预计算因子，估值类里有现成的 EP、BP、SP、PEG 等。用 `get_all_factors()` 查全部因子名。它的好处是已做好时间对齐，适合快速回测；下面的代码示例用 `get_fundamentals` 自己算，便于你完全掌控口径。

## 三、因子逐项说明

### 1. 市盈率族 — PE / EP

定义：PE = 总市值 / 归母净利润（TTM）。EP = 1/PE = 盈利收益率（earnings yield）。

聚宽来源：`valuation.pe_ratio`。

为什么用倒数 EP：亏损公司 PE 为负，直接排序会把"巨亏股"和"昂贵股"混在一起。取倒数后，亏损股的 EP 是个小负数，自然落在排名最低端，符合"基本面差"的直觉。**估值类因子统一建议转成倒数使用。**

方向：A 股长期价值效应下，EP 越高（越便宜）预期收益越高。

### 2. 市净率族 — PB / BP

定义：PB = 总市值 / 归母净资产。BP = 1/PB = 账面市值比。

聚宽来源：`valuation.pb_ratio`。

BP 是最经典的价值因子，Fama-French 三因子里的 HML 就是按 BP 排序构建的。注意净资产为负的公司 PB 会失真，需剔除。

方向：BP 越高越好。

### 3. 市销率族 — PS / SP

定义：PS = 总市值 / 营业收入（TTM）。SP = 1/PS。

聚宽来源：`valuation.ps_ratio`。

优点是营收比利润稳定、不易为负，所以对**亏损股、周期股、早期成长股**比 PE 更可用。缺点是忽略了盈利能力，低毛利行业的 SP 天然偏高，必须做行业中性化。

方向：SP 越高越好。

### 4. 市现率族 — PCF / CFP

定义：PCF = 总市值 / 经营活动现金流净额（TTM）。CFP = 1/PCF。

聚宽来源：`valuation.pcf_ratio`。

现金流比利润更难操纵，CFP 与利润口径的 EP 配合使用，可以交叉验证盈利质量。它和你已经在做的"现金流因子"是衔接点——如果把分母换成自由现金流，就得到 FCF 收益率。

方向：CFP 越高越好。

### 5. 企业价值族 — EV/EBITDA、EV/Sales、EV/EBIT

定义：EV（企业价值）= 总市值 + 有息负债 − 货币资金。常用倍数为 EV/EBITDA、EV/营业收入、EV/EBIT。

相比 PE，EV 类指标把债务考虑进来，**在不同资本结构的公司之间更可比**，尤其适合重资产、高杠杆行业。

聚宽来源（需自行拼装）：

- 总市值：`valuation.market_cap` × 1e8（换算成元，因为它单位是亿元）
- 货币资金：`balance.cash_equivalents`
- 有息负债：`balance.shortterm_loan` + `balance.longterm_loan` + `balance.bonds_payable` + `balance.non_current_liability_in_one_year`
- EBITDA ≈ `income.operating_profit` + 财务费用 `income.financial_expense` + 折旧摊销（现金流量表：`cash_flow.fixed_assets_depreciation` + `cash_flow.intangible_assets_amortization` + `cash_flow.defferred_expense_amortization`）

注意单位一致性（market_cap 是亿元、报表项是元），以及 income/cash_flow 是单季度值、算 TTM 要滚动四个季度。EBITDA 的口径在业内有多个版本，固定一个并保持一致即可。

方向：倒数（EBITDA/EV 等）越高越好。

### 6. 股息率（Dividend Yield）

定义：股息率 = 近 12 个月每股现金分红 / 当前股价。

聚宽来源：`valuation` 表无直接字段，从分红表计算。用 `finance.run_query(query(finance.STK_XR_XD)...)` 取除权除息记录，把除息日落在过去 12 个月内的每股现金分红求和，再除以当日股价。

它是深度价值与红利策略的交集——高股息往往同时意味着低估值、稳定现金流和股东友好。可进一步衍生"综合股东回报率"，归到股东回报因子那一类。

方向：越高越好（但要警惕"高股息陷阱"：股价暴跌也会被动推高股息率）。

### 7. 复合与相对估值

- **PEG** = PE / (净利润同比增速 × 100)。把成长性引入估值，PEG < 1 常被视为"成长被低估"。净利润增速取 `indicator.inc_net_profit_year_on_year`。增速为负或接近零时 PEG 会失真，需单独处理。
- **估值历史分位数**：PE 或 PB 在该股自身过去 3～5 年分布中的分位。回答的是"相对它自己贵不贵"，能弱化行业间天然差异。
- **估值同比变化**：ΔPE、ΔPB。估值的*变动方向*本身可作因子——估值持续被压缩有时是反转信号。

## 四、标准变换：从一个指标到一族因子

"穷举"的实质不是找新指标，而是对每个核心指标系统地做几种标准变换，每个指标因此扩展成一族：

| 变换 | 含义 | 例子 |
|---|---|---|
| 原始值 | 直接使用 | PE |
| 倒数 | 处理负值与右偏 | EP = 1/PE |
| 行业中性化 | 剔除行业天然差异 | 行业内 EP 排名 |
| 市值中性化 | 剔除大小盘影响 | 对 market_cap 回归取残差 |
| 历史分位 | 与自身历史比 | PE 五年分位 |
| 同比变化 | 时间维度 | ΔBP |
| 分位数离散化 | 转成分组打分 | EP 十分位 |

一个"市盈率"指标，经过这些变换就能产生七八个可独立检验的因子。五大类都按这个模式展开，因子数量自然成规模。

## 五、可直接运行的批量获取代码

下面的函数在给定日期按中证1000编制方案选股，并用估值因子调整权重，之后和官方的中证1000指数进行对比。(聚宽环境回测)

这个回测真正在回答的问题是:"在中证1000这个股票池里,按估值便宜程度调整权重,能不能跑赢纯市值加权的中证1000指数?" 跑出来的超额收益(策略 − 基准),就是估值因子在中证1000成分股上贡献的 alpha。(测的不是"用估值因子选股",而是"用估值因子调权重"。选股范围完全由指数编制规则决定,因子只动权重、不踢人)

```python
# -*- coding: utf-8 -*-
from jqdata import *
from jqdata import finance
import pandas as pd
import numpy as np
import datetime
"""
基准：官方中证1000指数
样本空间：中证全指近似
剔除规则：剔除中证800、剔除过去一年日均总市值前300、剔除成交金额后20%
调仓日期：每年6月、12月第二个星期五的下一交易日
样本数量：1000只
缓冲区：新样本前800优先进入，老样本前1200优先保留
换手限制：每次最多调整100只
权重：以总市值权重为基础，再按估值因子得分做权重倾斜
"""

def initialize(context):
    # 基准仍然用官方中证1000，用来对比
    g.benchmark_code = '000852.XSHG'      # 中证1000
    g.all_share_code = '000985.XSHG'      # 中证全指，作为样本空间近似
    g.csi800_code = '000906.XSHG'         # 中证800，用于剔除

    # 按中证1000编制方案，目标样本数为1000
    g.stock_num = 1000

    # 中证1000缓冲区与调整比例
    g.max_adjust_ratio = 0.10
    g.max_change_num = int(g.stock_num * g.max_adjust_ratio)  # 最多调整100只
    g.new_enter_rank = 800                                    # 新样本前800优先进入
    g.old_keep_rank = 1200                                    # 老样本前1200优先保留

    # 因子增强权重参数
    # 0.30 表示在市值权重基础上做适度因子倾斜
    g.factor_tilt_strength = 0.30
    g.min_tilt = 0.50
    g.max_tilt = 1.50

    # 为了运行速度，过去一年日均总市值用月度采样近似
    # 日均成交金额仍然用过去一年日频 money 计算
    g.market_cap_sample_step = 21
    g.has_rebalanced = False

    set_benchmark(g.benchmark_code)
    set_option('use_real_price', True)

    # 手续费
    set_order_cost(OrderCost(
        open_tax=0,
        close_tax=0.001,
        open_commission=0.0003,
        close_commission=0.0003,
        close_today_commission=0,
        min_commission=0
    ), type='stock')

    # 滑点
    set_slippage(FixedSlippage(0))

    log.set_level('order', 'error')

    # 每天检查，只有到中证1000定期调整日才真正调仓
    run_daily(rebalance, time='14:50')


def rebalance(context):
    today = context.current_dt.date()
    
    # 第一次允许直接建仓；之后只在中证1000定期调整日调仓
    if g.has_rebalanced and (not is_csi1000_regular_adjust_day(today)):
        return

    date = context.previous_date

    # 1. 按中证1000编制方案生成近似1000只样本
    sample_list = get_csi1000_like_universe(context, date)

    if len(sample_list) == 0:
        log.info('样本池为空，跳过调仓')
        return

    # 2. 计算估值因子分数
    factor_df = get_selected_valuation_factors(date, sample_list)

    if factor_df.empty:
        log.info('因子数据为空，跳过调仓')
        return

    # 3. 在1000只样本内做因子增强权重
    target_weights = get_factor_tilt_weights(sample_list, factor_df)

    if len(target_weights) == 0:
        log.info('目标权重为空，跳过调仓')
        return

    log.info('调仓日: %s, 目标持仓数量: %d' % (str(today), len(target_weights)))

    # 4. 卖出不在目标样本中的股票
    for stock in list(context.portfolio.positions.keys()):
        if stock not in target_weights.index:
            order_target_value(stock, 0)

    # 5. 按目标权重买入/调整
    total_value = context.portfolio.total_value
    
    for stock, weight in target_weights.items():
        target_value = total_value * weight
        order_target_value(stock, target_value)
        
    g.has_rebalanced = True
        

def is_csi1000_regular_adjust_day(today):
    """
    判断今天是否为中证1000定期调整日：
    每年6月和12月第二个星期五的下一交易日。
    """

    if today.month not in [6, 12]:
        return False

    year = today.year
    month = today.month

    first_day = datetime.date(year, month, 1)

    # weekday: Monday=0, Friday=4
    days_to_friday = (4 - first_day.weekday()) % 7
    first_friday = first_day + datetime.timedelta(days=days_to_friday)
    second_friday = first_friday + datetime.timedelta(days=7)

    # 取第二个星期五之后的第一个交易日
    trade_days = get_trade_days(
        start_date=second_friday + datetime.timedelta(days=1),
        end_date=second_friday + datetime.timedelta(days=15)
    )

    if len(trade_days) == 0:
        return False

    target_day = pd.Timestamp(trade_days[0]).date()

    return today == target_day


def get_csi1000_like_universe(context, date):
    """
    按中证1000编制方案近似构建样本：
    1. 中证全指作为样本空间
    2. 剔除中证800
    3. 剔除过去一年日均总市值前300
    4. 剔除过去一年日均成交金额后20%
    5. 按过去一年日均总市值选1000只
    6. 定期调整使用800/1200缓冲区和10%调整限制

    注意：
    为了在聚宽回测里跑得动，过去一年日均总市值用约每21个交易日采样一次近似。
    """

    end_date = date
    start_date = date - datetime.timedelta(days=370)

    # 1. 中证全指样本空间
    all_stocks = get_index_stocks(g.all_share_code, date=date)
    all_stocks = filter_stock(all_stocks)
    
    if len(all_stocks) == 0:
        return []
    
    # 2. 先在中证全指样本空间内计算过去一年成交金额和总市值
    metrics_all = get_one_year_liquidity_mcap(all_stocks, start_date, end_date)
    
    if metrics_all.empty:
        return []
    
    metrics_all = metrics_all.dropna(subset=['avg_money', 'avg_market_cap'])
    
    if metrics_all.empty:
        return []
    
    # 3. 剔除中证800
    csi800 = set(get_index_stocks(g.csi800_code, date=date))
    
    # 4. 剔除过去一年日均总市值前300名
    top300_big = set(
        metrics_all.sort_values('avg_market_cap', ascending=False).head(300).index
    )
    
    universe = [
        s for s in all_stocks
        if s not in csi800 and s not in top300_big and s in metrics_all.index
    ]
    
    if len(universe) == 0:
        return []
    
    # 后面直接用 metrics_all 里的数据，不要重新算
    metrics = metrics_all.loc[universe].copy()

    if metrics.empty:
        return []

    metrics = metrics.dropna(subset=['avg_money', 'avg_market_cap'])

    if metrics.empty:
        return []

    # 5. 成交金额排名
    metrics = metrics.sort_values('avg_money', ascending=False)
    metrics['money_rank'] = np.arange(1, len(metrics) + 1)
    n = len(metrics)

    current_holdings = list(context.portfolio.positions.keys())
    old_set = set(current_holdings)

    # 新样本：剔除成交金额后20%，即保留前80%
    new_liquid = metrics[metrics['money_rank'] <= int(n * 0.80)].index

    # 老样本：缓冲区，成交金额排名前90%仍可进入下一步
    old_liquid = metrics[
        (metrics.index.isin(old_set)) &
        (metrics['money_rank'] <= int(n * 0.90))
    ].index

    eligible = list(set(new_liquid).union(set(old_liquid)))

    metrics = metrics.loc[eligible].copy()

    if metrics.empty:
        return []

    # 6. 按过去一年日均总市值排名
    metrics = metrics.sort_values('avg_market_cap', ascending=False)
    metrics['mcap_rank'] = np.arange(1, len(metrics) + 1)

    # 第一次建仓：直接选前1000
    if len(current_holdings) == 0:
        return list(metrics.head(g.stock_num).index)

    # 老样本：市值排名前1200优先保留
    old_keep = [
        s for s in current_holdings
        if s in metrics.index and int(metrics.loc[s, 'mcap_rank']) <= g.old_keep_rank
    ]

    # 如果老样本太多，只保留排名最好的1000只
    old_keep = sorted(
        old_keep,
        key=lambda x: metrics.loc[x, 'mcap_rank']
    )[:g.stock_num]

    # 为了控制调整比例，一般最多换100只
    # 所以至少尽量保留900只老样本
    if len(old_keep) < g.stock_num - g.max_change_num:
        old_extra = [
            s for s in current_holdings
            if s in metrics.index and s not in old_keep
        ]

        old_extra = sorted(
            old_extra,
            key=lambda x: metrics.loc[x, 'mcap_rank']
        )

        need_old = g.stock_num - g.max_change_num - len(old_keep)
        old_keep = old_keep + old_extra[:need_old]

    need_num = g.stock_num - len(old_keep)

    if need_num <= 0:
        return old_keep[:g.stock_num]

    # 新样本：市值排名前800优先进入
    new_candidates_800 = [
        s for s in metrics.index
        if s not in old_keep
        and s not in old_set
        and int(metrics.loc[s, 'mcap_rank']) <= g.new_enter_rank
    ]

    # 如果前800不够，再按市值排名补足
    other_candidates = [
        s for s in metrics.index
        if s not in old_keep
        and s not in new_candidates_800
    ]

    add_list = (new_candidates_800 + other_candidates)[:need_num]

    sample_list = old_keep + add_list

    return sample_list[:g.stock_num]
    
def get_one_year_liquidity_mcap(stock_list, start_date, end_date):
    """
    计算过去一年：
    - avg_money：日均成交金额
    - avg_market_cap：日均总市值近似

    成交金额用 get_price 日频数据。
    总市值为了速度，用约每21个交易日采样一次 get_fundamentals 近似。
    """

    result = pd.DataFrame(index=stock_list)

    # 1. 过去一年日均成交金额
    money_list = []

    for batch in split_list(stock_list, 300):
        try:
            price_df = get_price(
                batch,
                start_date=start_date,
                end_date=end_date,
                frequency='daily',
                fields=['money'],
                panel=False
            )

            if price_df is not None and len(price_df) > 0:
                money_list.append(price_df)

        except Exception as e:
            log.info('get_price money error: %s' % str(e))

    if len(money_list) == 0:
        result['avg_money'] = np.nan
    else:
        money_df = pd.concat(money_list, axis=0)

        if 'code' in money_df.columns:
            result['avg_money'] = money_df.groupby('code')['money'].mean()
        else:
            result['avg_money'] = np.nan

    # 2. 过去一年日均总市值近似：每21个交易日采样一次
    trade_days = get_trade_days(start_date=start_date, end_date=end_date)

    if len(trade_days) == 0:
        result['avg_market_cap'] = np.nan
        return result

    sample_days = trade_days[::g.market_cap_sample_step]

    if len(sample_days) == 0:
        sample_days = [end_date]

    cap_list = []

    for d in sample_days:
        try:
            q = query(
                valuation.code,
                valuation.market_cap
            ).filter(
                valuation.code.in_(stock_list)
            )

            cap = get_fundamentals(q, date=d)

            if cap is not None and len(cap) > 0:
                cap = cap.drop_duplicates('code').set_index('code')
                cap_list.append(cap['market_cap'])

        except Exception as e:
            log.info('get_fundamentals market_cap error: %s' % str(e))

    if len(cap_list) == 0:
        result['avg_market_cap'] = np.nan
    else:
        cap_df = pd.concat(cap_list, axis=1)
        result['avg_market_cap'] = cap_df.mean(axis=1)

    return result

def get_factor_tilt_weights(sample_list, factor_df):
    """
    在中证1000近似样本的1000只股票内做因子增强权重。

    基础权重：总市值权重
    因子增强：score 越高，权重越高
    """

    df = factor_df.reindex(sample_list).copy()

    # 如果有个别股票因子缺失，给中性分数0
    df['score'] = pd.to_numeric(df['score'], errors='coerce').fillna(0)

    # market_cap 单位是亿元，只用于权重比例，不需要转成元
    df['market_cap'] = pd.to_numeric(df['market_cap'], errors='coerce')

    df = df.dropna(subset=['market_cap'])
    df = df[df['market_cap'] > 0]

    if df.empty:
        return pd.Series(dtype=float)

    # 基础市值权重
    base_weight = df['market_cap'] / df['market_cap'].sum()

    # 因子倾斜系数
    tilt = 1.0 + g.factor_tilt_strength * df['score']
    tilt = tilt.clip(lower=g.min_tilt, upper=g.max_tilt)

    weight = base_weight * tilt
    weight = weight / weight.sum()

    return weight


def filter_stock(stock_list):
    current_data = get_current_data()
    result = []

def get_selected_valuation_factors(date, stock_list):
    """
    只保留用户选出的估值因子：
    PE、PB、PS、PCF、EV/EBITDA、EV/Sales、Dividend_Yield

    score 方向：
    PE / PB / PS / PCF / EV_EBITDA / EV_Sales 越小越好
    Dividend_Yield 越大越好
    """

    if len(stock_list) == 0:
        return pd.DataFrame()

    # 1. 取 valuation 表中的估值因子
    q = query(
        valuation.code,
        valuation.pe_ratio,
        valuation.pb_ratio,
        valuation.ps_ratio,
        valuation.pcf_ratio,
        valuation.market_cap
    ).filter(
        valuation.code.in_(stock_list)
    )

    val = get_fundamentals(q, date=date)

    if val is None or len(val) == 0:
        return pd.DataFrame()

    val = val.drop_duplicates('code').set_index('code')

    raw = pd.DataFrame(index=val.index)
    raw['market_cap'] = val['market_cap']
    raw['PE'] = val['pe_ratio']
    raw['PB'] = val['pb_ratio']
    raw['PS'] = val['ps_ratio']
    raw['PCF'] = val['pcf_ratio']

    # 2. 计算 EV
    ev = get_enterprise_value(date, list(raw.index), val['market_cap'])

    # 3. 计算 TTM Sales 和 TTM EBITDA
    ttm = get_ttm_sales_ebitda(date, list(raw.index))

    raw['EV_EBITDA'] = ev / ttm['EBITDA_TTM']
    raw['EV_Sales'] = ev / ttm['Sales_TTM']

    # 4. 计算近 12 个月股息率
    raw['Dividend_Yield'] = get_dividend_yield(date, list(raw.index), val['market_cap'])

    # 5. 清洗：倍数类必须大于 0，否则容易把亏损股误判为“便宜”
    multiple_cols = ['PE', 'PB', 'PS', 'PCF', 'EV_EBITDA', 'EV_Sales']

    for col in multiple_cols:
        raw[col] = pd.to_numeric(raw[col], errors='coerce')
        raw.loc[raw[col] <= 0, col] = np.nan

    raw['Dividend_Yield'] = pd.to_numeric(raw['Dividend_Yield'], errors='coerce')
    raw.loc[raw['Dividend_Yield'] < 0, 'Dividend_Yield'] = np.nan
    raw = raw.replace([np.inf, -np.inf], np.nan)

    # 6. 标准化 + 方向调整
    # 倍数类越小越好，所以乘 -1
    # 股息率越大越好，所以乘 +1
    direction = {
        'PE': -1,
        'PB': -1,
        'PS': -1,
        'PCF': -1,
        'EV_EBITDA': -1,
        'EV_Sales': -1,
        'Dividend_Yield': 1
    }

    score_df = pd.DataFrame(index=raw.index)

    for col, sign in direction.items():
        score_df[col] = sign * clean_zscore(raw[col])

    raw['valid_factor_count'] = score_df.notna().sum(axis=1)
    raw['score'] = score_df.mean(axis=1)
    
    # 做指数增强时不强行剔除样本，因子不足的股票给中性分数0
    raw.loc[raw['valid_factor_count'] < 4, 'score'] = 0
    raw['score'] = raw['score'].fillna(0)
    
    return raw


def get_enterprise_value(date, stock_list, market_cap_series):
    """
    EV = 总市值 + 有息负债 - 货币资金
    market_cap 单位：亿元，所以要乘 1e8 转成元
    """

    q = query(
        balance.code,
        balance.cash_equivalents,
        balance.shortterm_loan,
        balance.longterm_loan,
        balance.bonds_payable,
        balance.non_current_liability_in_one_year
    ).filter(
        balance.code.in_(stock_list)
    )

    bal = get_fundamentals(q, date=date)

    if bal is None or len(bal) == 0:
        return pd.Series(np.nan, index=stock_list)

    bal = bal.drop_duplicates('code').set_index('code').reindex(stock_list)

    cols = [
        'cash_equivalents',
        'shortterm_loan',
        'longterm_loan',
        'bonds_payable',
        'non_current_liability_in_one_year'
    ]

    for col in cols:
        bal[col] = pd.to_numeric(bal[col], errors='coerce').fillna(0)

    interest_debt = (
        bal['shortterm_loan']
        + bal['longterm_loan']
        + bal['bonds_payable']
        + bal['non_current_liability_in_one_year']
    )

    cash = bal['cash_equivalents']

    market_cap_yuan = market_cap_series.reindex(stock_list) * 1e8

    ev = market_cap_yuan + interest_debt - cash
    ev[ev <= 0] = np.nan

    return ev

def get_ttm_sales_ebitda(date, stock_list):
    """
    聚宽回测可运行版本。

    注意：
    原本严格 EBITDA = operating_profit + financial_expense + 折旧摊销
    但你的聚宽回测环境没有 cash_flow.fixed_assets_depreciation 等字段，
    所以这里改成 EBITDA 近似值：
    EBITDA_proxy = operating_profit + financial_expense

    这个值更接近 EBIT proxy，但可以保证回测先跑通。
    """

    fields = [
        income.total_operating_revenue,
        income.operating_profit,
        income.financial_expense
    ]

    all_df = []

    for batch in split_list(stock_list, 200):
        try:
            df = get_history_fundamentals(
                security=batch,
                fields=fields,
                watch_date=date,
                count=4,
                interval='1q'
            )

            if df is not None and len(df) > 0:
                all_df.append(df)

        except Exception as e:
            log.info('get_history_fundamentals error: %s' % str(e))

    if len(all_df) == 0:
        return pd.DataFrame(index=stock_list, columns=['Sales_TTM', 'EBITDA_TTM'])

    hist = pd.concat(all_df, axis=0)

    need_cols = [
        'total_operating_revenue',
        'operating_profit',
        'financial_expense'
    ]

    for col in need_cols:
        if col not in hist.columns:
            hist[col] = 0
        hist[col] = pd.to_numeric(hist[col], errors='coerce').fillna(0)

    grouped = hist.groupby('code')

    result = pd.DataFrame(index=grouped.size().index)

    result['Sales_TTM'] = grouped['total_operating_revenue'].sum()

    result['EBITDA_TTM'] = (
        grouped['operating_profit'].sum()
        + grouped['financial_expense'].sum()
    )

    result = result.reindex(stock_list)

    result.loc[result['Sales_TTM'] <= 0, 'Sales_TTM'] = np.nan
    result.loc[result['EBITDA_TTM'] <= 0, 'EBITDA_TTM'] = np.nan

    return result

def get_dividend_yield(date, stock_list, market_cap_series):
    """
    股息率 = 近 12 个月现金分红总额 / 当前总市值

    finance.STK_XR_XD.bonus_amount_rmb 单位通常为万元
    valuation.market_cap 单位为亿元
    所以分母 market_cap * 10000 后也是万元
    """

    end_date = date
    start_date = end_date - datetime.timedelta(days=365)

    dividend = pd.Series(0.0, index=stock_list)

    all_df = []

    for batch in split_list(stock_list, 1000):
        q = query(
            finance.STK_XR_XD.code,
            finance.STK_XR_XD.a_xr_date,
            finance.STK_XR_XD.bonus_amount_rmb
        ).filter(
            finance.STK_XR_XD.code.in_(batch),
            finance.STK_XR_XD.a_xr_date >= start_date,
            finance.STK_XR_XD.a_xr_date <= end_date,
            finance.STK_XR_XD.bonus_amount_rmb > 0
        )

        df = finance.run_query(q)

        if df is not None and len(df) > 0:
            all_df.append(df)

    if len(all_df) == 0:
        return dividend

    div_df = pd.concat(all_df, axis=0)
    div_sum = div_df.groupby('code')['bonus_amount_rmb'].sum()

    cap_wan = market_cap_series.reindex(div_sum.index) * 10000.0

    dy = div_sum / cap_wan
    dividend.loc[dy.index] = dy

    dividend = dividend.replace([np.inf, -np.inf], np.nan).fillna(0.0)

    return dividend


def clean_zscore(s):
    """
    MAD 去极值 + z-score 标准化
    """

    s = pd.to_numeric(s, errors='coerce')
    s = s.replace([np.inf, -np.inf], np.nan)

    median = s.median()
    mad = (s - median).abs().median()

    if pd.isnull(mad) or mad == 0:
        lower = s.quantile(0.01)
        upper = s.quantile(0.99)
    else:
        lower = median - 3 * 1.4826 * mad
        upper = median + 3 * 1.4826 * mad

    s = s.clip(lower=lower, upper=upper)

    std = s.std()

    if pd.isnull(std) or std == 0:
        return pd.Series(np.nan, index=s.index)

    return (s - s.mean()) / std


def split_list(lst, n):
    for i in range(0, len(lst), n):
        yield lst[i:i + n]
```

## 六、因子方向与检验提示

每个因子都要用 IC（因子值与下期收益的截面相关系数）逐期验证，不能只看一次回测收益。重点看：IC 均值与正负、ICIR（IC 均值 / IC 标准差，衡量稳定性）、分层组合收益的单调性、多空组合的年化收益与最大回撤、换手率。

估值因子普遍换手率低、单调性较好，但有明显的**风格周期**——在成长占优的市场里价值因子会长期跑输，这是因子特性而非因子失效。检验时务必拉长样本、覆盖牛熊。可用聚宽自带的因子分析功能或 Alphalens 完成。

---

## 七、注意事项与坑

- **负值处理**：PE / PS / PCF 在亏损时为负。统一用倒数（EP/SP/CFP）解决，或显式剔除负值股。
- **单位**：`market_cap` / `circulating_market_cap` 单位是亿元，和报表项（元）拼算 EV 时要换算。
- **TTM vs 累计**：`valuation` 的比率已是 TTM；但自己从 income/cash_flow 算指标时，那些是单季度值，要滚动四季求和。
- **未来函数**：直接用 `valuation` 字段较安全；一旦自己从财务报表算因子，必须用 `pubDate`（公告日）而非 `statDate`（报告期）来判断"当时是否可见"。
- **行业中性化几乎必做**：银行天然低 PB、科技天然高 PE，不做行业中性，因子会变成"行业选择"而非"估值选择"。
- **极端右偏**：估值因子分布严重右偏，去极值 + 标准化是必要步骤。
- **高股息陷阱**：股息率被动走高可能源于股价暴跌，需结合盈利质量一起看。
